# Best Practices

## Принципы разработки

### SOLID

- **S**ingle Responsibility: каждый модуль/класс отвечает за одну задачу
- **O**pen/Closed: открыт для расширения, закрыт для модификации
- **L**iskov Substitution: подтипы должны заменять базовые типы
- **I**nterface Segregation: много специализированных интерфейсов лучше одного общего
- **D**ependency Inversion: зависимость от абстракций, а не от конкретных реализаций

### KISS (Keep It Simple, Stupid)

- Пиши простой и понятный код
- Избегай излишней сложности
- Решай задачу минимальными средствами

### DRY (Don't Repeat Yourself)

- Не дублируй код
- Выноси повторяющуюся логику в переиспользуемые функции/компоненты
- Используй композицию и наследование разумно

### FSD (Feature-Sliced Design)

Основная архитектура приложения:

```
src/
├── app/          # Инициализация приложения, провайдеры, роутинг
├── pages/        # Страницы приложения
├── widgets/      # Крупные самостоятельные блоки
├── features/     # Функциональные части (действия пользователя)
├── entities/     # Бизнес-сущности
├── shared/       # Переиспользуемый код (ui, lib, api)
```

## Дополнительные правила

- Используй TypeScript для типобезопасности
- Пиши чистые функции где возможно
- Избегай глубокой вложенности (max 3 уровня)
- Называй переменные и функции понятно и однозначно
- Комментируй только сложную бизнес-логику, код должен быть самодокументируемым
