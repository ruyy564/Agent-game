# Глобальный стор

## Когда использовать глобальный стор

Используй глобальный стор только в следующих случаях:

- Данные используются в 3+ несвязанных компонентах на разных уровнях дерева
- Состояние нужно сохранять между размонтированием компонентов
- Сложная бизнес-логика требует централизованного управления
- Данные пользователя (авторизация, профиль)
- Глобальные настройки приложения
- Кэш данных, используемых по всему приложению

## Когда НЕ использовать глобальный стор

- Локальное состояние компонента (используй useState)
- Состояние формы (используй react-hook-form или локальный стейт)
- Данные, передаваемые только между родителем и ребенком (используй props)
- Серверное состояние (используй react-query/tanstack-query)
- URL параметры (используй react-router)

## Технология

- Используй только **Zustand** для глобального стора
- Не используй Redux, MobX или другие библиотеки управления состоянием

## Структура стора

Стор должен находиться в папке `entities` и содержать модульную структуру:

```
src/entities/<entity-name>/
├── model/
│   ├── store.ts       # Zustand стор
│   ├── types.ts       # TypeScript типы
│   └── selectors.ts   # Селекторы для оптимизации
└── index.ts           # Публичный API
```

## Подключение стора

- Стор **всегда** подключается через провайдер React
- Провайдер создается в `app/providers`
- Инициализация стора происходит в провайдере

Пример:

```tsx
// app/providers/StoreProvider.tsx
import { createContext, useContext } from 'react';
import { createStore } from 'zustand';

const StoreContext = createContext(null);

export const StoreProvider = ({ children }) => {
  const store = createStore(...);
  return <StoreContext.Provider value={store}>{children}</StoreContext.Provider>;
};

export const useStore = () => useContext(StoreContext);
```

## Правила написания стора

- Разделяй стор на слайсы по доменам
- Используй селекторы для оптимизации ре-рендеров
- Не храни в сторе вычисляемые значения
- Экшены должны быть чистыми функциями
- Избегай вложенных объектов - используй нормализацию данных
