# React Best Practices

## Refs

- Используй refs для доступа к DOM-узлам и хранения значений, которые не влияют на рендеринг
- Не читай и не изменяй ref.current во время рендеринга
- Используй useRef для значений, которые не должны вызывать ре-рендер

## Effects

- Effects запускаются после коммита на экран
- Используй Effects только для синхронизации с внешними системами (API, браузерные API, сторонние библиотеки)
- Не используй Effects для трансформации данных для рендеринга - делай это во время рендеринга
- Не используй Effects для обработки пользовательских событий - используй обработчики событий
- Всегда указывай массив зависимостей
- Возвращай cleanup функцию из Effect для очистки ресурсов

## Оптимизация

- Используй useMemo для кэширования дорогих вычислений
- Используй useCallback для мемоизации функций, передаваемых в дочерние компоненты
- Не оптимизируй преждевременно - сначала измерь производительность

## Custom Hooks

- Выноси повторяющуюся логику в кастомные хуки
- Называй хуки с префиксом "use"
- Кастомные хуки должны быть чистыми и переиспользуемыми

## Правила хуков

- Вызывай хуки только на верхнем уровне компонента
- Не вызывай хуки внутри условий, циклов или вложенных функций
- Вызывай хуки только из React-компонентов или кастомных хуков

## Избегай

- Не используй Effects для управления состоянием, которое можно вычислить из пропсов или стейта
- Не создавай цепочки Effects, обновляющих стейт друг за другом
- Не подписывайся на события в Effects без cleanup функции
