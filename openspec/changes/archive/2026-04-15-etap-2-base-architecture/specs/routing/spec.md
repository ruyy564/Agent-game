## ADDED Requirements

### Requirement: Навигация между экранами

Система SHALL обеспечивать маршрутизацию между 4 основными экранами игры с использованием React Router v7.

#### Scenario: Переход на стартовый экран

- **WHEN** пользователь открывает приложение по URL `/`
- **THEN** система отображает стартовый экран (StartScreen)

#### Scenario: Переход к выбору миссии

- **WHEN** пользователь переходит по URL `/missions`
- **THEN** система отображает экран выбора миссии (MissionSelect)

#### Scenario: Переход к игровому экрану

- **WHEN** пользователь переходит по URL `/game/:missionId` с валидным missionId
- **THEN** система отображает игровой экран (GameScreen) с загруженной миссией

#### Scenario: Переход к экрану результатов

- **WHEN** пользователь переходит по URL `/results`
- **THEN** система отображает экран результатов (ResultsScreen)

### Requirement: Параметры маршрута

Система SHALL поддерживать динамические параметры в URL для передачи идентификаторов миссий.

#### Scenario: Извлечение missionId из URL

- **WHEN** пользователь находится на `/game/email-phishing-1`
- **THEN** система извлекает параметр `missionId` со значением `email-phishing-1`

#### Scenario: Доступ к параметрам в компоненте

- **WHEN** компонент GameScreen рендерится
- **THEN** компонент MUST иметь доступ к параметру `missionId` через React Router hooks

### Requirement: Lazy loading страниц

Система SHALL использовать lazy loading для всех страниц для оптимизации начальной загрузки.

#### Scenario: Отложенная загрузка страницы

- **WHEN** пользователь впервые переходит на маршрут `/missions`
- **THEN** система загружает код компонента MissionSelect только в момент перехода

#### Scenario: Кэширование загруженных страниц

- **WHEN** пользователь повторно переходит на уже посещенный маршрут
- **THEN** система использует закэшированный код компонента без повторной загрузки

### Requirement: Программная навигация

Система SHALL предоставлять API для программной навигации между маршрутами.

#### Scenario: Навигация из обработчика события

- **WHEN** пользователь нажимает кнопку "Начать игру" на StartScreen
- **THEN** система программно переходит на маршрут `/missions`

#### Scenario: Навигация с параметрами

- **WHEN** пользователь выбирает миссию с id `code-review-1`
- **THEN** система программно переходит на маршрут `/game/code-review-1`

### Requirement: Обработка несуществующих маршрутов

Система SHALL обрабатывать переходы на несуществующие маршруты.

#### Scenario: Переход на несуществующий маршрут

- **WHEN** пользователь переходит по URL `/nonexistent`
- **THEN** система перенаправляет на стартовый экран `/`
