## ADDED Requirements

### Requirement: Управление очками игрока

Система SHALL хранить и управлять текущим количеством очков игрока.

#### Scenario: Начальное значение очков

- **WHEN** игрок начинает новую игру
- **THEN** система устанавливает `score` в 0

#### Scenario: Добавление очков

- **WHEN** игрок правильно идентифицирует угрозу стоимостью 100 очков
- **THEN** система увеличивает `score` на 100

#### Scenario: Вычитание очков за ошибку

- **WHEN** игрок неправильно идентифицирует угрозу
- **THEN** система уменьшает `score` на 50 (штраф)

#### Scenario: Получение текущих очков

- **WHEN** компонент запрашивает очки через селектор
- **THEN** система возвращает текущее значение `score`

### Requirement: Управление уровнем игрока

Система SHALL хранить текущий уровень сложности, на котором играет игрок.

#### Scenario: Установка уровня

- **WHEN** игрок выбирает уровень сложности "Middle Analyst"
- **THEN** система устанавливает `currentLevel` в `LevelDifficulty.MIDDLE`

#### Scenario: Получение текущего уровня

- **WHEN** компонент запрашивает уровень через селектор
- **THEN** система возвращает значение `currentLevel`

#### Scenario: Уровень по умолчанию

- **WHEN** игрок впервые запускает игру
- **THEN** система устанавливает `currentLevel` в `LevelDifficulty.JUNIOR`

### Requirement: Управление достижениями

Система SHALL хранить список разблокированных достижений игрока.

#### Scenario: Добавление достижения

- **WHEN** игрок выполняет условие для достижения "First Blood"
- **THEN** система добавляет `first-blood` в массив `achievements`

#### Scenario: Проверка наличия достижения

- **WHEN** компонент проверяет, есть ли у игрока достижение `first-blood`
- **THEN** система возвращает `true`, если достижение в массиве `achievements`

#### Scenario: Получение всех достижений

- **WHEN** компонент запрашивает все достижения через селектор
- **THEN** система возвращает массив идентификаторов достижений

### Requirement: Управление статистикой игрока

Система SHALL хранить статистику игрока (пройденные миссии, найденные угрозы, точность).

#### Scenario: Увеличение счетчика пройденных миссий

- **WHEN** игрок успешно завершает миссию
- **THEN** система увеличивает `stats.missionsCompleted` на 1

#### Scenario: Увеличение счетчика найденных угроз

- **WHEN** игрок правильно идентифицирует угрозу
- **THEN** система увеличивает `stats.threatsFound` на 1

#### Scenario: Увеличение счетчика ошибок

- **WHEN** игрок неправильно идентифицирует угрозу
- **THEN** система увеличивает `stats.mistakes` на 1

#### Scenario: Вычисление точности

- **WHEN** компонент запрашивает точность через селектор
- **THEN** система вычисляет и возвращает `threatsFound / (threatsFound + mistakes) * 100`

#### Scenario: Получение общей статистики

- **WHEN** компонент запрашивает статистику через селектор
- **THEN** система возвращает объект `stats` с полями `missionsCompleted`, `threatsFound`, `mistakes`

### Requirement: Сброс прогресса игрока

Система SHALL предоставлять функцию для сброса прогресса игрока.

#### Scenario: Полный сброс прогресса

- **WHEN** игрок выбирает "Начать заново" в настройках
- **THEN** система сбрасывает `score`, `achievements`, `stats` в начальные значения

#### Scenario: Сохранение уровня при сбросе

- **WHEN** игрок сбрасывает прогресс
- **THEN** система сохраняет `currentLevel` без изменений
