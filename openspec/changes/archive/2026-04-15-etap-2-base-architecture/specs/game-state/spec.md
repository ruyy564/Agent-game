## ADDED Requirements

### Requirement: Управление текущей миссией

Система SHALL хранить и управлять состоянием текущей активной миссии.

#### Scenario: Установка текущей миссии

- **WHEN** пользователь выбирает миссию с id `email-phishing-1`
- **THEN** система сохраняет `currentMissionId` со значением `email-phishing-1` в глобальном сторе

#### Scenario: Получение текущей миссии

- **WHEN** компонент запрашивает текущую миссию через селектор
- **THEN** система возвращает объект миссии с полными данными

#### Scenario: Очистка текущей миссии

- **WHEN** пользователь завершает миссию или возвращается к выбору миссий
- **THEN** система устанавливает `currentMissionId` в `null`

### Requirement: Управление игровым режимом

Система SHALL хранить текущий активный режим игры (email, code, logs).

#### Scenario: Установка режима игры

- **WHEN** пользователь переключается на вкладку "Code Review"
- **THEN** система устанавливает `currentMode` в `GameMode.CODE_REVIEW`

#### Scenario: Получение текущего режима

- **WHEN** компонент запрашивает текущий режим через селектор
- **THEN** система возвращает значение `currentMode`

#### Scenario: Режим по умолчанию

- **WHEN** пользователь начинает новую миссию
- **THEN** система устанавливает `currentMode` в `GameMode.EMAIL_ANALYSIS`

### Requirement: Управление таймером

Система SHALL управлять игровым таймером для отслеживания времени прохождения миссии.

#### Scenario: Запуск таймера

- **WHEN** пользователь начинает миссию
- **THEN** система запускает таймер с начальным значением 0 секунд

#### Scenario: Обновление таймера

- **WHEN** таймер активен
- **THEN** система увеличивает значение `elapsedTime` каждую секунду

#### Scenario: Пауза таймера

- **WHEN** пользователь ставит игру на паузу
- **THEN** система останавливает увеличение `elapsedTime`

#### Scenario: Сброс таймера

- **WHEN** пользователь завершает миссию
- **THEN** система сбрасывает `elapsedTime` в 0

### Requirement: Управление состоянием паузы

Система SHALL управлять состоянием паузы игры.

#### Scenario: Постановка на паузу

- **WHEN** пользователь нажимает кнопку паузы
- **THEN** система устанавливает `isPaused` в `true`

#### Scenario: Снятие с паузы

- **WHEN** пользователь нажимает кнопку продолжить
- **THEN** система устанавливает `isPaused` в `false`

#### Scenario: Пауза по умолчанию

- **WHEN** пользователь начинает новую миссию
- **THEN** система устанавливает `isPaused` в `false`

### Requirement: Управление найденными угрозами

Система SHALL отслеживать список найденных угроз в текущей миссии.

#### Scenario: Добавление найденной угрозы

- **WHEN** пользователь правильно идентифицирует угрозу с id `threat-1`
- **THEN** система добавляет `threat-1` в массив `foundThreats`

#### Scenario: Получение списка найденных угроз

- **WHEN** компонент запрашивает найденные угрозы через селектор
- **THEN** система возвращает массив идентификаторов найденных угроз

#### Scenario: Очистка найденных угроз

- **WHEN** пользователь начинает новую миссию
- **THEN** система очищает массив `foundThreats`

### Requirement: Сброс состояния игры

Система SHALL предоставлять функцию для сброса всего игрового состояния.

#### Scenario: Сброс при начале новой миссии

- **WHEN** пользователь начинает новую миссию
- **THEN** система сбрасывает `elapsedTime`, `foundThreats`, `isPaused` в начальные значения

#### Scenario: Сброс при выходе из игры

- **WHEN** пользователь возвращается к выбору миссий
- **THEN** система сбрасывает `currentMissionId`, `currentMode` и все игровое состояние
