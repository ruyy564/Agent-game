## Why

Текущее приложение не имеет маршрутизации, глобального управления состоянием и типизированных игровых сущностей. Без базовой архитектуры невозможно реализовать навигацию между экранами, сохранение прогресса игрока и координацию между компонентами. Этап 2 закладывает фундамент для всей игры.

## What Changes

- Настройка React Router v7 для навигации между 4 основными экранами (стартовый, выбор миссии, игра, результаты)
- Создание Zustand стора с провайдером для управления состоянием игрока и игры
- Определение TypeScript типов для всех игровых сущностей (угрозы, уровни, миссии, режимы)
- Создание структуры провайдеров в app/providers (StoreProvider, RouterProvider)
- Интеграция провайдеров в корневой App компонент

## Capabilities

### New Capabilities

- `routing`: Маршрутизация между экранами игры с поддержкой параметров и lazy loading
- `game-state`: Глобальное управление состоянием игры (текущая миссия, режим, таймер, пауза)
- `player-state`: Управление состоянием игрока (очки, уровень, достижения, статистика)
- `game-types`: Типизация игровых сущностей (ThreatType, LevelDifficulty, MissionType, GameMode)
